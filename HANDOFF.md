## Рекомендация по разбиению index.html (ответ на вопрос пользователя)

**Короткий ответ:** да, разбить стоит — но это задача для Phase 0 следующей сессии (10–15 минут работы), не сейчас.

**Почему стоит:**
- Сейчас `index.html` ~1300 строк ≈ 38–40k токенов на полный read. На каждой правке слайда читается лишнее.
- Правка одного слайда не должна показывать дифф по всему файлу.
- Меньшие файлы = быстрее grep, меньше шансов ошибиться в `Edit`.

**Как разбить (рекомендуемая схема):**

```
slides/
├── _head.html         ← <head>, <link>, <script>, SVG icon sprite
├── _stage-open.html   ← <body>, <deck-stage width="1920" height="1080">
├── 01-cover.html
├── 02-company.html
├── 03-products.html
├── 04-clients.html
├── 05-why.html
├── 06-omnichannel.html
├── 07-dialog.html
├── 08-ai-tools.html
├── 09-ai-attributes.html
├── 10-ai-prompter.html
├── 11-savings.html
├── 12-audit.html
├── 13-flexibility.html
├── 14-security.html       ← новый
├── 15-pricing.html        ← объединение 14+16 старых
├── 16-calculator.html     ← новый
├── 17-roadmap.html        ← новый
├── 18-contacts.html
└── _stage-close.html  ← </deck-stage>, inline <style>, </body></html>
```

И простой Node-скрипт `build.js`:

```js
const fs = require('fs');
const path = require('path');
const dir = 'slides';
const files = ['_head.html', '_stage-open.html',
  ...fs.readdirSync(dir).filter(f => /^\d{2}-/.test(f)).sort(),
  '_stage-close.html'];
const out = files.map(f => fs.readFileSync(path.join(dir, f), 'utf8')).join('\n');
fs.writeFileSync('index.html', out);
console.log(`Built index.html from ${files.length} fragments`);
```

Запуск: `node build.js`. Можно повесить `npm run build` или делать руками после правок.

**Цена:**
- Один build-step (один npm-скрипт без зависимостей).
- `index.html` становится сгенерированным артефактом — добавить в `.gitignore` или продолжать коммитить (так Chrome открывает его без сборки).
- Каждая правка — `node build.js` после редактирования слайда.

**Альтернатива без build-шага:** оставить один `index.html`, но добавить чёткие маркеры:

```html
<!-- ========== SLIDE 07 START ========== -->
<section ...> ... </section>
<!-- ========== SLIDE 07 END ========== -->
```

Тогда `Grep "SLIDE 07 START"` → `Read offset+limit` = быстрый таргетный read без full-file.

**Предлагаемое решение:** в начале следующей сессии я делаю Phase 0 — разбиваю на slides/*.html + build.js, коммичу. С этого момента каждая правка — read 1 файла на 100–200 строк вместо 1300.
