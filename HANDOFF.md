## Следующая итерация — I-3 (с чего начинать новую сессию)

### Скоуп
Слайды **07 «Диалоговое окно оператора»** и **12 «Раздел Аудит»** — оба используют живые HTML-макеты `screens/dialogs.html` и `screens/audit.html`, плюс hotspot-демо.

### Концепт hotspot-демо
- Real screen в центре слайда (через `<iframe src="screens/dialogs.html" style="..." sandbox>`).
- 5–6 hotspot-областей по координатам поверх iframe (абсолютно позиционированные `<div class="hotspot">`).
- Цикл (`data-hotspot-cycle`, шаг 3500 ms): по очереди подсвечиваются области + рядом всплывает описание (`<div class="hotspot-caption">`).
- Слайд 07: 6 областей — написание текста / ИИ-суфлёр / ИИ-инструменты / переназначение / фильтры и сортировка / атрибуты диалога.
- Слайд 12: 5 областей — фильтры / поиск / общая сумма (12 847) / детализация диалога / экспорт (Excel + .docx).

### Технический паттерн
Движок hotspot-cycle уже реализован в `animations.js` (функция `initHotspotCycles`). Стили — в `animations.css` (`.hotspot`, `.hotspot-caption`). Нужно только:
1. Сверстать iframe-контейнер.
2. Подогнать координаты hotspot'ов под реальный layout скрина (открыть `screens/dialogs.html` в Chrome, замерить через DevTools).
3. Написать тексты для caption'ов.

### Тонкости
- iframe нужно скейлить (real screen в исходном размере → масштабировать в карточку слайда). Проще всего: `transform: scale(0.7)` + `transform-origin: top left` + контейнер с `width:1430px; height:855px; overflow:hidden` (чтобы scrollbars не торчали).
- Если iframe тяжёлый — рассмотреть `loading="lazy"` или замену на статический скриншот PNG в этой версии и обновление до iframe позже.
- Скрины в `screens/dialogs.html` тянут CSS с `chat-dev.robotmia.ru`. Если нет интернета — стили могут не применяться. Уточнить у пользователя, нужна ли offline-версия.

## Рекомендация по разбиению index.html (ответ на вопрос пользователя)

**Короткий ответ:** да, разбить стоит — но это задача для Phase 0 следующей сессии (10–15 минут работы), не сейчас.

**Почему стоит:**
- Сейчас `index.html` ~1300 строк ≈ 38–40k токенов на полный read. На каждой правке слайда читается лишнее.
- Правка одного слайда не должна показывать дифф по всему файлу.
- Меньшие файлы = быстрее grep, меньше шансов ошибиться в `Edit`.

**Как разбить (рекомендуемая схема):**

```
slides/
├── _head.html         ← <head>, <link>, <script>, SVG icon sprite
├── _stage-open.html   ← <body>, <deck-stage width="1920" height="1080">
├── 01-cover.html
├── 02-company.html
├── 03-products.html
├── 04-clients.html
├── 05-why.html
├── 06-omnichannel.html
├── 07-dialog.html
├── 08-ai-tools.html
├── 09-ai-attributes.html
├── 10-ai-prompter.html
├── 11-savings.html
├── 12-audit.html
├── 13-flexibility.html
├── 14-security.html       ← новый
├── 15-pricing.html        ← объединение 14+16 старых
├── 16-calculator.html     ← новый
├── 17-roadmap.html        ← новый
├── 18-contacts.html
└── _stage-close.html  ← </deck-stage>, inline <style>, </body></html>
```

И простой Node-скрипт `build.js`:

```js
const fs = require('fs');
const path = require('path');
const dir = 'slides';
const files = ['_head.html', '_stage-open.html',
  ...fs.readdirSync(dir).filter(f => /^\d{2}-/.test(f)).sort(),
  '_stage-close.html'];
const out = files.map(f => fs.readFileSync(path.join(dir, f), 'utf8')).join('\n');
fs.writeFileSync('index.html', out);
console.log(`Built index.html from ${files.length} fragments`);
```

Запуск: `node build.js`. Можно повесить `npm run build` или делать руками после правок.

**Цена:**
- Один build-step (один npm-скрипт без зависимостей).
- `index.html` становится сгенерированным артефактом — добавить в `.gitignore` или продолжать коммитить (так Chrome открывает его без сборки).
- Каждая правка — `node build.js` после редактирования слайда.

**Альтернатива без build-шага:** оставить один `index.html`, но добавить чёткие маркеры:

```html
<!-- ========== SLIDE 07 START ========== -->
<section ...> ... </section>
<!-- ========== SLIDE 07 END ========== -->
```

Тогда `Grep "SLIDE 07 START"` → `Read offset+limit` = быстрый таргетный read без full-file.

**Предлагаемое решение:** в начале следующей сессии я делаю Phase 0 — разбиваю на slides/*.html + build.js, коммичу. С этого момента каждая правка — read 1 файла на 100–200 строк вместо 1300.
